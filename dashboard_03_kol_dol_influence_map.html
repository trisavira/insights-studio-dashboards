Data model. The grain is the actor (kol_id). Core fields: kol_id · name · institution · tier · type(KOL/DOL/both) · specialty · geography · influence_score (+ component breakdown) · traditional_score · digital_score · stance · stance_trend · engaged_flag · last_engagement · msl_owner · npse · topics[] · pubs · citations · trial_roles · guideline_roles, plus an edge table (kol_a · kol_b · tie_type · weight) for the network and a referral table (referrer · receiver · volume). The hard upstream work is identity resolution (one real person = one kol_id across publications, congress, social, Veeva) and the influence model + graph centrality / community detection — Tableau renders these but does not compute them. PII / consent governance applies throughout — all individuals here are illustrative. ~30 worksheets across 8 sections; selecting a KOL drives the 360 view.
